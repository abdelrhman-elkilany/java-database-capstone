# User Stories for Smart Clinic Management System

This document outlines the user stories for the Smart Clinic Management System. The system includes three main types of users: Doctor, Patient, and Admin. Each user type has specific needs and functionalities they can access. Below are the defined user stories for each user role.

---

## Doctor User Stories

### 1. View Patient Appointments
**As a doctor**, I want to view patient appointments so that I can prepare for consultations.

- **Acceptance Criteria:**
  - The system shows a list of upcoming appointments for the doctor.
  - Appointments include relevant details such as patient name, date, and time.

### 2. Update Patient Records
**As a doctor**, I want to update patient records so that the medical information is up-to-date.

- **Acceptance Criteria:**
  - The system allows the doctor to access a patient's medical record.
  - The doctor can add notes, prescriptions, or update diagnosis information.
  
### 3. Search for Patients
**As a doctor**, I want to search for a patient by name or ID so that I can quickly access their records.

- **Acceptance Criteria:**
  - The system provides a search functionality to filter patients based on their name or ID.

---

## Patient User Stories

### 1. Book an Appointment
**As a patient**, I want to book an appointment with a doctor so that I can receive medical consultation.

- **Acceptance Criteria:**
  - The system displays available doctors and their time slots.
  - The patient can select a doctor and choose an available time slot to schedule an appointment.

### 2. View Medical History
**As a patient**, I want to view my medical history so that I can monitor my health progress.

- **Acceptance Criteria:**
  - The system shows a list of past consultations, diagnoses, and treatments for the patient.

### 3. Receive Appointment Reminders
**As a patient**, I want to receive reminders for my upcoming appointments so that I do not miss them.

- **Acceptance Criteria:**
  - The system sends automated reminders via email or SMS to patients about their upcoming appointments.

---

## Admin User Stories

### 1. Manage Doctors' Availability
**As an admin**, I want to manage doctors' availability so that patients can book appointments with available doctors.

- **Acceptance Criteria:**
  - The system allows the admin to set or update the working hours of each doctor.

### 2. Manage Patient Records
**As an admin**, I want to manage patient records so that the clinic has accurate data for all patients.

- **Acceptance Criteria:**
  - The system allows the admin to add, update, or delete patient records.
